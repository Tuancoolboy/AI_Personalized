## Current Truth

- The app is **Next.js 16.2.7 App Router**, not Vite.
- Runtime stack from `package.json`: React 19.2.4, TypeScript, Tailwind CSS v4, shadcn/ui, Recharts, Supabase SSR/client, OpenAI SDK.
- Deploy target is **Vercel** with Framework Preset **Next.js**.
- Tailwind v4 is configured through `src/frontend/app/globals.css`, `@tailwindcss/postcss`, and shadcn imports. There is no Tailwind v3-style `tailwind.config.js`.
- Supabase supports two modes:
  - **Demo mode:** when `NEXT_PUBLIC_SUPABASE_URL` and the public Supabase key are missing, auth/app flows use demo cookies and localStorage.
  - **Real mode:** when Supabase env vars exist, auth/session behavior should use Supabase.
- Supabase public key compatibility: the app now accepts `NEXT_PUBLIC_SUPABASE_PUBLISHABLE_KEY` first and falls back to legacy `NEXT_PUBLIC_SUPABASE_ANON_KEY` so newer Supabase dashboard projects work without refactoring auth/session code.
- Register flow intentionally redirects back to `/login?registered=1` after success and signs out any session Supabase created, so users always log in explicitly before onboarding.
- Supabase employee persistence work is governed by `specs/BE-07-supabase-employee-persistence.md`; read it before touching Supabase auth, clients, migrations, RLS, profiles, progress, quiz, time logs, leads, or chat usage.
- Manager and multi-manager production work is governed by `specs/BE-08-multi-manager-roadmap.md`; read it before touching manager auth, manager dashboards, employee management, organization/team schema, cross-user RLS, or manager APIs.
- Broad Phase 2 product work is governed by `specs/PHASE2-SPEC.md`; it covers company creation/login, custom departments and job roles, company-authored paths, four AI capabilities, Aha Moment/community, leaderboards, and AI cost controls.
- Company invite link work is governed by `specs/BE-10-company-invite-links.md`; read it before touching token-only invite links, `/moi/[token]`, manager invite-link APIs, or register/login redirects from invite links.
- Clarify-card parsing now strips markdown links before counting question marks, so lesson links such as `[AI là gì? ...]` do not get misread as a new clarify prompt.
- Employee coach prompt now explicitly answers progress questions like "đang học đến đâu" and next-step questions like "nên học gì tiếp" directly instead of converting them into `__CLARIFY__`; the chat cache now short-circuits those next-step questions before OpenAI so they cannot fall back to a clarify card; feedback-like messages such as "câu trả lời sao rồi", "quá chung", "rút gọn lại", "thêm ví dụ", "đúng nhưng chưa đủ sâu", "lệch role", or "so sánh benchmark/kỳ trước" now trigger a repair path that prefers a direct rewrite over a new clarify card.

- Backend native chat helpers now mirror the same greeting stripping, and draft chat history returns `coreContext: null` so the FE/BE paths stay aligned on fresh-session boundaries.
