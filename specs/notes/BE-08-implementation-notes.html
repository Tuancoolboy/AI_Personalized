3. Tradeoff AI phải cân nhắc

RLS manager policy vs service-role aggregation: vẫn tạo RLS cho membership, nhưng team aggregation dùng service role trong Route Handler vì cần đọc profiles, module_progress, và quiz_results của nhiều user.

Email đã tồn tại: tìm user bằng Admin API theo email, đọc profiles.full_name, profiles.phone_number và profiles.role_id nếu có, rồi upsert membership. Nếu tài khoản chưa đăng ký, API trả lỗi để nhân viên tạo tài khoản trước.

Department field: dùng lại role IDs kebab-case (kinh-doanh, ke-toan...) để tránh thêm taxonomy mới.

Real-mode empty team: khi Supabase đã cấu hình, dashboard/list không fallback về TEAM_MEMBERS demo nữa. Manager mới sẽ thấy trạng thái rỗng cho đến khi có membership thật trong organization_members.

Manager self-row: API team loại row của manager đang đăng nhập khỏi danh sách/trang dashboard để trưởng phòng xem nhân viên khác, không tự xuất hiện như một nhân viên cần quản lý.

Private organization per manager: checkbox Cấp quyền quản lý hiện tạo organization riêng cho email được cấp quyền, thay vì đưa người đó vào công ty hiện tại của manager đang thao tác.

Single organization per account: API thêm nhân viên và accept link mời đều kiểm tra mọi membership của user trước khi ghi. Nếu user đã thuộc organization khác, hệ thống trả conflict hoặc hiển thị lỗi link mời thay vì tự chuyển công ty.

Reset dev data: thêm script SQL riêng để xóa dữ liệu dev/test, bao gồm auth.users. Không chạy script này trên production.

4. Những điều khác bạn nên biết

  • Migration cần chạy: chạy 0008_multi_manager_core.sql, sau đó 20260610110000_manager_private_organizations.sql để tách manager khỏi Tổ chức mặc định, rồi 20260610120000_single_organization_membership.sql để enforce một user một công ty.
  • Schema cũ: 0008 có đoạn compatibility để nâng cấp bảng từng tạo nháp với cột department sang department_id và thêm unique constraint organization_id,user_id cho Supabase upsert.
  • Full name khi đăng ký: chạy thêm 0009_profile_full_name_metadata.sql để trigger lấy full_name từ Supabase Auth metadata; manager add form không còn yêu cầu nhập họ tên.
  • Số điện thoại: chạy thêm 0010_profile_phone_number.sql; manager team API có thể đọc profiles.phone_number để hiển thị dưới email nhân viên, nhưng add form không yêu cầu số điện thoại.
  • Known limitation: chưa có UI chuyển công ty; nếu cần chuyển nhân viên từ công ty A sang công ty B thì phải làm thao tác admin/manual có chủ đích.
  • Follow-up: chạy smoke test với vuhaituan@gmail.com, ronaldo36@gmail.com, và một employee như mixi@gmail.com để xác nhận công ty thứ hai bị chặn.
