BE-09 Leads API — Implementation Notes

Ngày: 09/06/2026 · Phạm vi: GET leads + UI quản lý (bản sơ khai US-07)

1. Quyết định AI tự ra (ngoài spec)

  • Thêm lib/manager-auth.ts — detect quản lý qua email pattern (quanly@, manager@…) cho cả demo mode và Supabase Auth.
  • GET đọc leads bằng service role server-side; không thêm RLS SELECT cho user (tránh lộ lead ra client anon).
  • Trang /quan-ly/leads redirect nhân viên về /lo-trinh; API trả 403 nếu không phải manager.
  • Login Supabase: email quản lý redirect thẳng /quan-ly (giống demo).

2. Thay đổi so với spec gốc

  • US-07 / GD2-01 ghi Phase 2 — đã ship sớm chỉ phần xem leads, không phải dashboard đội thật.
  • BE-09 ban đầu chỉ POST — mở rộng GET + UI manager.

3. Tradeoff

  • Email pattern vs profiles.user_type: nhanh cho GĐ1, dễ demo; GĐ2 nên thay bằng cột DB + policy.
  • Service role read vs RLS manager policy: service role đơn giản, không expose SELECT; tradeoff là logic quản lý nằm ở app layer.
  • Demo mode không Supabase: GET trả mảng rỗng + message — lead chỉ persist khi có DB.

4. Lưu ý khác

  • Dashboard /quan-ly vẫn dùng mock team-data.ts; chỉ card/bảng leads là dữ liệu thật.
  • Follow-up GĐ2: export CSV, thông báo lead mới, profiles.user_type, bảo vệ toàn bộ /quan-ly/* ở server.
