Gate 3 Submission — Implementation Notes

Ngày cập nhật: 25/06/2026

1. Quyết định AI tự ra ngoài spec

  • Dùng duy nhất c2-app-009.vercel.app làm production URL.
  • Tách evidence thành submission, eval, guardrails, cost và demo plan.
  • Thêm script tổng hợp baseline để metrics có thể tái tạo.
  • Dùng ba cost scenarios và 25% contingency thay vì một con số duy nhất.

2. Những chỗ thay đổi so với spec gốc

  • Phép tính cũ 20 request/user/tháng được thay bằng low/base/high usage.
  • Input-token estimate tăng từ 500 lên 3.000 cho tutor vì runtime hiện có nhiều context block.
  • Không coi file eval backend placeholder là evidence chính; dùng raw eval của flow /tro-ly.

3. Tradeoff đã cân nhắc

  • Không chạy thêm paid OpenAI eval khi chưa có xác nhận riêng; dùng baseline thật đã lưu và production cache probe.
  • Không chuyển custom domain trong Gate 3 vì cần DNS/domain ownership; chỉ ghi rõ migration target.
  • Cost report dùng assumptions bảo thủ vì streaming path chưa xuất exact token usage.

4. Lưu ý và follow-up

  • Cần con người quay voice-over và upload video mới.
  • Nên chạy lại live eval authenticated trên production trước submission nếu budget cho phép.
  • Sau Gate 3 cần persist token/cost usage và đo time-to-first-token.
