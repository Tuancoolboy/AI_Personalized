Quality gate

Kiểm thử và Definition of Done

Mỗi phase phải khóa correctness theo tenant trước khi chuyển sang phase tiếp theo. Build pass không thay thế được RLS và browser smoke.

Ma trận kiểm thử bắt buộc

  • Unit: scoring, permission helper, rubric parser, cost, sensitive-data scan.
  • API: company creation, OAuth redirect, CRUD tenant scope, grading, reflection, budget.
  • RLS: own employee, same-org peer, manager, other org, anonymous, service role.
  • Browser: owner setup → invite; employee join → learn → grade → Aha; manager analytics.
  • Isolation: Manager A tuyệt đối không đọc được Company B.

Một task chỉ “Done” khi

  • Migration có rollback section và không phá dữ liệu.
  • RLS/authorization được test theo matrix.
  • UI copy tiếng Việt hoàn chỉnh.
  • Deterministic fallback hoạt động khi không có OpenAI.
  • AI output được validate bằng schema.
  • Token-consuming operation được ghi usage.
  • Worklog, continuation, checklist và notes được cập nhật.
  • npm run lint, npm run test, npm run build và smoke test pass.
