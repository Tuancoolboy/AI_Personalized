# Docs Structure

`docs/` chỉ dành cho tài liệu tham khảo dài hạn. Đây **không** phải nơi đặt spec source of truth hay sprint planning.

## Canonical folders

- `docs/ops/` — setup, deploy, Supabase, OAuth, vận hành môi trường
- `docs/product/` — user journey, product context, tài liệu mô tả sản phẩm không phải spec
- `docs/handoffs/` — bàn giao, tổng hợp kết quả, handoff giữa các đợt làm việc
- `docs/archive/` — tài liệu cũ, file nhập từ ngoài, tài liệu không còn là source of truth
- `docs/guide/` — guidebook, tài liệu học, template tham khảo

## Không đặt ở `docs/`

- Spec mới: đặt trong `specs/`
- Implementation notes: đặt trong `specs/notes/`
- Sprint plan / review / demo plan / decision memo: đặt trong `planning/`
- Task summary ngắn: đặt trong `tasks/`
